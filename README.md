## 📁 Estructura del Repositorio

El repositorio está organizado de manera secuencial según los entregables e incrementos del proyecto, guiados por la metodología **Scrum++**:

### 📄 [Documento 0](./Documento%200/)
Contiene la fase de levantamiento, análisis inicial y planificación del proyecto.
* **Grupo 17 - Documento 0.docx:** Informe formal con la definición del proyecto, objetivos y alcance.
* **Matriz de Requerimientos versus Casos de Uso:** Archivos de control para mantener la trazabilidad entre las necesidades del usuario (UR) y las funcionalidades del sistema (RF).

### 🚀 [Incremento 1](./Incremento%201/)
Contiene la documentación formal y los artefactos generados durante el primer ciclo de desarrollo técnico.
* **Entrega del Incremento 1.docx:** Informe detallado que incluye la justificación de la arquitectura, la pila del producto (Product Backlog), Casos de Prueba de QA y la gestión del proyecto mediante ceremonias Scrum.
* **Presentación Incremento 1.pdf:** Material de apoyo visual utilizado para la defensa del incremento, destacando los casos demostrativos (CU14 y CU30).

#### 📂 [Anexos (Diagramas y Modelos)](./Incremento%201/Anexos/)
Directorio que almacena las exportaciones gráficas de la arquitectura y el diseño del sistema:
* 🧩 `Diagrama_de_Componentes.drawio.png`: Mapeo de la arquitectura cliente-servidor (Capa de Vista, Controladores, Adaptadores y Base de Datos).
* 🌐 `Diagrama_de_Despliegue.drawio.png`: Topología de infraestructura y servicios externos.
* 📊 `MERE.drawio.png`: Modelo Entidad-Relación Extendido del dominio de datos.
* 🗄️ `Modelo_Físico.drawio.png`: Diseño relacional optimizado para la base de datos MySQL.
* 📱 `Árbol de Navegación.png`: Mapa de pantallas y rutas de la interfaz de usuario (Paciente/Profesional/Admin).

## ⚙️ Repositorio de Código
Este repositorio es exclusivo para el respaldo de la **Documentación Técnica**. El código fuente de la aplicación (Frontend en React Native y Backend en Node.js/Express) se mantiene y versiona en el repositorio principal del proyecto.
